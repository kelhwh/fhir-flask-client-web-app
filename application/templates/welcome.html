Welcome!

This is a FHIR client web app built with Flask.

Intro

In the world of information, sharing is important. But it's extremely difficult to communicate when it comes to complex domains like healthcare. Even the definition of a disease can vary across regions, let alone the format of medical records. Fortunately, there is a popular interoperability standard, FHIR, to facilitate the exchange of health information.

This web app connects to a public FHIR testing server, everything is rendered on the fly (except for the login information). That being said, the app can be easily migrated to any FHIR server by changing the server the app connects to. This is exactly the potential of FHIR!


Scope

Currently, appointment history is redered and the app can only read the result. All FHIR resources are actually possible to render and requests are not limited to read but also create, update and delete. I aim to build the appointments and medication page with both read and update operations.


Instruction

  1. Login with prefilled credentials, if failed, go register the user first.
  2. Go to Appointments, where you can see the appointment history of the patient fetched from the FHIR server.

Note

  • The registration is verified with the patient information from the FHIR server, any change to the prefilled values will fail the registration. You won't be able to register a random user unless you know what users are in the server.


Disclaimer

The app connects to the HAPI FHIR Reference Server with STU3 endpoint http://hapi.fhir.org/baseDstu3/. All information display is retrieved from the server and is for testing purpose.
